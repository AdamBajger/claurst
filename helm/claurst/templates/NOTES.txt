1. Wait for the external IP to be assigned (LoadBalancer):

     kubectl get svc {{ include "claurst.fullname" . }} \
       --namespace {{ .Release.Namespace }} \
       --watch

2. Once the EXTERNAL-IP is available, connect:

     ssh -p {{ .Values.service.port }} claurst@<EXTERNAL-IP>

   The claurst TUI starts immediately.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  If you did not supply API keys at install time you can configure them
  interactively from inside claurst with the /connect command, or upgrade
  the release:

    helm upgrade {{ .Release.Name }} ./helm/claurst \
      --namespace {{ .Release.Namespace }} \
      --reuse-values \
      --set apiKeys.anthropic=sk-ant-...
